Basic Implementation

The basic implementation of the download attribute, is well, basic. Very basic. The reason why this section is smaller and on its own, is to also hint at the large number of restrictions of the download attribute and HTML as well. The standard call for making an object a link to data that is downloaded by the user is:

The a tag is the same as before, however the download attribute is placed after the href declaration. This tells the browser running that the object enclosed in the a brackets will be downloaded by the user. This statement is the default statement, and simply downloads the exact file as is to the users download directory, while keeping the user on the same page. A variation, and in fact the only variation to the download attribute is below:

Setting the download attribute with a new string name, will change the name of the file once it is downloaded to by the user. This is useful when the data being accessed has an obscure naming convention by a third party, or if the source location requires a unique naming convention for organization. In short, these are the only two modifications to the download attribute. Not very good if you want your website to do more and have these features across multiple web platforms. Download is currently is only supported in HTML with these browsers: Chrome 14.0+ FireFox 20.0+ Opera 15+
